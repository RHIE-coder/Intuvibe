# Interactive Mode — 키보드 단축키, 입력 모드, 인터랙티브 기능

---

`?`를 누르면 현재 환경의 단축키 목록 확인 가능.
macOS에서 Option/Alt 키 단축키 사용 시 터미널에서 **Option을 Meta로 설정** 필요.

## 1. 키보드 단축키

### 일반 제어

| 단축키 | 동작 |
|--------|------|
| `Ctrl+C` | 입력/생성 취소 |
| `Ctrl+D` | 세션 종료 (EOF) |
| `Ctrl+G` / `Ctrl+X Ctrl+E` | 외부 텍스트 에디터로 프롬프트 편집 |
| `Ctrl+L` | 프롬프트 입력 삭제 (대화 이력은 유지) |
| `Ctrl+O` | 상세 출력 토글 (도구 사용 상세 + MCP 호출 확장) |
| `Ctrl+R` | 명령 이력 역방향 검색 |
| `Ctrl+V` / `Cmd+V` / `Alt+V` | 클립보드 이미지 붙여넣기 → `[Image #N]` 칩 삽입 |
| `Ctrl+B` | 실행 중인 태스크 백그라운드 전환 (tmux에서 2번) |
| `Ctrl+T` | 태스크 리스트 토글 |
| `Ctrl+X Ctrl+K` | 백그라운드 에이전트 전체 종료 (3초 내 2번으로 확인) |
| `←/→` | 대화상자 탭 순환 |
| `↑/↓` | 명령 이력 탐색 |
| `Esc Esc` | 이전 지점으로 되감기 또는 요약 |
| `Shift+Tab` / `Alt+M` | 권한 모드 순환 |
| `Alt+P` | 모델 전환 (프롬프트 유지) |
| `Alt+T` | 확장 사고 모드 토글 |
| `Alt+O` | fast mode 토글 |

### 텍스트 편집

| 단축키 | 동작 |
|--------|------|
| `Ctrl+K` | 커서→줄 끝 삭제 (붙여넣기 가능) |
| `Ctrl+U` | 커서→줄 시작 삭제 (반복 시 여러 줄 삭제) |
| `Ctrl+Y` | 삭제한 텍스트 붙여넣기 |
| `Alt+Y` | `Ctrl+Y` 후 삭제 이력 순환 |
| `Alt+B` / `Alt+F` | 단어 단위 이동 (뒤/앞) |

### 멀티라인 입력

| 방법 | 단축키 |
|------|--------|
| 퀵 이스케이프 | `\` + `Enter` (모든 터미널) |
| macOS 기본 | `Option+Enter` |
| Shift+Enter | iTerm2, WezTerm, Ghostty, Kitty에서 기본 동작. 그 외 `/terminal-setup` |
| 제어 문자 | `Ctrl+J` |
| 붙여넣기 모드 | 코드 블록/로그 직접 붙여넣기 |

### 퀵 커맨드

| 접두사 | 동작 |
|--------|------|
| `/` | 명령 또는 스킬 실행 |
| `!` | Bash 모드 (직접 셸 명령 실행) |
| `@` | 파일 경로 자동완성 |

## 2. Bash 모드 (`!` 접두사)

```
! npm test
! git status
```

- Claude의 해석/승인 **불필요** — 직접 실행
- 명령 + 출력이 **대화 컨텍스트에 추가**됨
- `Ctrl+B`로 백그라운드 전환 가능
- **이력 기반 자동완성**: 부분 입력 + `Tab` → 이전 `!` 명령에서 완성
- 빈 프롬프트에 `!`로 시작하는 텍스트 붙여넣기 → 자동 bash 모드 진입
- `Escape`, `Backspace`, `Ctrl+U`로 빈 프롬프트에서 종료

## 3. `/btw` — 사이드 질문

```
/btw what was the name of that config file again?
```

대화 이력에 **추가되지 않는** 일회성 질문.

| 특성 | 내용 |
|------|------|
| **컨텍스트** | 현재 대화 전체 가시 |
| **도구 접근** | **없음** — 컨텍스트 내 정보만으로 답변 |
| **응답** | 단일 응답 (후속 턴 없음) |
| **비용** | 프롬프트 캐시 재사용으로 최소 |
| **병렬** | Claude 작업 중에도 실행 가능 (메인 턴 미중단) |

`Space`, `Enter`, `Esc`로 답변 닫기.

> [insight] `/btw`는 서브에이전트의 **정반대**: 전체 대화 가시성 + 도구 없음. 서브에이전트는 도구 있음 + 빈 컨텍스트. "이미 아는 것에 대해 물어보기"는 `/btw`, "새로운 것을 찾아오기"는 서브에이전트.

## 4. 프롬프트 제안

- 세션 시작 시: git 이력 기반 예시 명령 (최근 작업 파일 반영)
- 응답 후: 대화 이력 기반 후속 단계 제안
- `Tab` / `→` = 수락, `Enter` = 수락+전송, 타이핑 시작 = 무시
- 프롬프트 캐시 재사용으로 추가 비용 최소. 캐시 cold일 때 자동 스킵
- 대화 첫 턴 이후, 비인터랙티브, plan 모드에서 자동 스킵

비활성화:
```
CLAUDE_CODE_ENABLE_PROMPT_SUGGESTION=false
```

## 8. Vim 에디터 모드

`/config` → Editor mode로 활성화.

### 모드 전환

| 키 | 동작 | 출발 모드 |
|----|------|----------|
| `Esc` | NORMAL 진입 | INSERT |
| `i`/`I` | 커서 앞/줄 시작에서 삽입 | NORMAL |
| `a`/`A` | 커서 뒤/줄 끝에서 삽입 | NORMAL |
| `o`/`O` | 아래/위에 새 줄 | NORMAL |

### NORMAL 모드 네비게이션

`h`/`j`/`k`/`l`, `w`/`e`/`b`, `0`/`$`/`^`, `gg`/`G`, `f`/`F`/`t`/`T`{char}, `;`/`,`

커서가 입력 시작/끝에 있고 더 이동 불가 시 → **화살표 키가 명령 이력 탐색으로 전환**.

### NORMAL 모드 편집

`x`, `dd`/`D`, `dw`/`de`/`db`, `cc`/`C`, `cw`/`ce`/`cb`, `yy`/`Y`, `yw`/`ye`/`yb`, `p`/`P`, `>>`/`<<`, `J`, `.`

### 텍스트 오브젝트

`d`/`c`/`y` 연산자와 조합: `iw`/`aw`, `iW`/`aW`, `i"`/`a"`, `i'`/`a'`, `i(`/`a(`, `i[`/`a[`, `i{`/`a{`

## 9. 명령 이력

- 작업 디렉토리별 저장
- `/clear` 시 이력 리셋 (이전 세션 대화는 보존, `/resume`으로 재개 가능)
- `!` history expansion은 기본 비활성화

### `Ctrl+R` 역방향 검색

1. `Ctrl+R` → 검색 활성화
2. 텍스트 입력 → 매칭 결과에 검색어 하이라이트
3. `Ctrl+R` 반복 → 이전 매칭으로 이동
4. `Tab`/`Esc` = 수락 후 편집 계속, `Enter` = 수락 + 즉시 실행
5. `Ctrl+C` = 취소 + 원래 입력 복원, 빈 검색에서 `Backspace` = 취소

## 10. 트랜스크립트 뷰어

`Ctrl+O`로 토글. 열린 상태에서:

| 단축키 | 동작 |
|--------|------|
| `Ctrl+E` | 전체 내용 표시 토글 (리바인드: `transcript:toggleShowAll`) |
| `q` / `Ctrl+C` / `Esc` | 뷰어 종료 (리바인드: `transcript:exit`) |

> [insight] `Ctrl+O` 상세 출력 모드는 MCP 호출을 확장 표시한다. 기본적으로 "Queried slack" 한 줄로 축약되는 MCP 호출의 실제 내용을 볼 수 있다. MCP 도구 디버깅이나 하네스 개발 시 필수.

> [insight] 태스크 리스트는 컨텍스트 압축 후에도 유지된다. 이는 태스크 리스트가 컨텍스트 윈도우 외부에 별도 저장됨을 의미. `CLAUDE_CODE_TASK_LIST_ID`로 세션 간 공유하면 연속적인 작업 추적이 가능.
