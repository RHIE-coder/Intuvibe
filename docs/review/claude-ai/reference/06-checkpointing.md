# Checkpointing — 편집 추적, 되감기, 요약

---

Claude의 파일 편집을 자동 추적하여 이전 상태로 되감기 가능.
세션 레벨의 "로컬 Undo" — Git을 대체하지 않고 보완.

---

## 1. 동작 원리

- **매 유저 프롬프트마다** 새 체크포인트 자동 생성
- 파일 편집 도구(`Edit`, `Write`, `NotebookEdit`)의 변경만 추적
- 세션 간 **유지** (재개된 대화에서도 접근 가능)
- 30일 후 세션과 함께 자동 정리 (설정 변경 가능)

---

## 2. 되감기 & 요약

**진입**: `Esc Esc` 또는 `/rewind`

→ 세션의 프롬프트 목록 스크롤 → 지점 선택 → 액션 선택:

| 액션 | 코드 복원 | 대화 복원 | 설명 |
|------|:---------:|:---------:|------|
| **Restore code and conversation** | O | O | 코드 + 대화 모두 해당 지점으로 |
| **Restore conversation** | X | O | 현재 코드 유지, 대화만 되감기 |
| **Restore code** | O | X | 대화 유지, 코드만 되감기 |
| **Summarize from here** | X | 압축 | 선택 지점 이후를 AI 요약으로 교체 |
| **Never mind** | - | - | 취소 |

복원/요약 후 → 선택 메시지의 원래 프롬프트가 **입력 필드에 복원**되어 재전송/편집 가능.

---

## 3. Summarize vs Restore vs Compact

| | Restore | Summarize from here | `/compact` |
|---|---------|-------------------|------------|
| **대상** | 코드/대화 상태 되돌림 | 선택 지점 이후만 압축 | 전체 대화 압축 |
| **파일 변경** | 되돌림 | 없음 | 없음 |
| **이전 메시지** | 제거 | 그대로 유지 | 전체 요약 |
| **원본 보존** | 체크포인트 | 세션 트랜스크립트에 보존 | 소실 |
| **용도** | 실패한 시도 되돌리기 | 장황한 디버깅 중간부터 압축 | 전체 컨텍스트 절약 |

Summarize 시 선택적 지시문으로 **요약 초점 지정** 가능.

> 분기 시도가 목적이면 Summarize 대신 `claude --continue --fork-session`으로 원본 세션 보존한 채 분기.

---

## 4. 활용 시나리오

- **대안 탐색**: 다른 구현 접근법 시도 후 시작점으로 복귀
- **실수 복구**: 버그 도입한 변경 빠르게 되돌리기
- **반복 실험**: 작동하는 상태로 되돌리면서 변형 시도
- **컨텍스트 절약**: 장황한 디버깅의 중간부터 요약, 초기 지시문은 그대로 유지

---

## 5. 제한사항

### Bash 명령 변경 미추적

```bash
rm file.txt       # 추적 안 됨
mv old.txt new.txt  # 추적 안 됨
cp source.txt dest.txt  # 추적 안 됨
```

파일 편집 도구가 아닌 **Bash 도구로 수행된 파일 변경은 되감기 불가**.

### 외부 변경 미추적

- 사용자가 Claude Code 밖에서 직접 수정한 파일
- 다른 동시 세션의 편집
- 예외: 현재 세션과 **같은 파일**을 수정한 경우만 우연히 캡처

### Git 대체 아님

| 체크포인트 | Git |
|-----------|-----|
| 세션 레벨 로컬 Undo | 영구적 이력 |
| 빠른 복구 | 커밋, 브랜치, 협업 |
| 30일 자동 정리 | 영구 보존 |

> [insight] 체크포인트는 파일 편집 도구만 추적하고 Bash 도구의 파일 조작은 추적하지 않는다. `rm`, `mv`, `git checkout` 같은 Bash 명령의 파일 변경은 되감기 불가. 하네스에서 안전한 복구를 보장하려면 파일 조작을 가능한 한 Edit/Write 도구로 유도하거나, 중요 작업 전 Git 커밋을 강제하는 훅을 설치하는 것이 방어적 전략.

> [insight] "Summarize from here"는 `/compact`의 정밀 버전이다. 초기 지시문은 그대로 두고 중간의 장황한 디버깅 로그만 압축할 수 있다. 원본 메시지는 세션 트랜스크립트에 보존되어 Claude가 필요 시 참조 가능. 하네스에서 컨텍스트 관리 전략에 `/compact` 외에 선택적 요약을 추가하면 더 정밀한 컨텍스트 최적화가 가능.
