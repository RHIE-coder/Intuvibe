# Agent Skills Overview

---

모듈형 확장 기능. 지시문·메타데이터·리소스(스크립트, 템플릿)를 패키징해 Claude가 자동으로 활용. 대화 수준 프롬프트와 달리 온디맨드 로딩.

- ZDR: ❌ 미지원
- 필요 베타 헤더 (API): `code-execution-2025-08-25`, `skills-2025-10-02`, `files-api-2025-04-14`

---

## 1. 3단계 Progressive Disclosure 로딩

| 레벨 | 로딩 시점 | 토큰 비용 | 내용 |
|------|---------|---------|------|
| Level 1: 메타데이터 | 항상 (시작 시) | ~100 tokens/Skill | YAML frontmatter (`name`, `description`) |
| Level 2: 지시문 | 트리거 시 | 5k 이하 | SKILL.md 본문 |
| Level 3: 리소스/코드 | 필요 시 | 사실상 무제한 | 번들 파일 (bash 실행, 컨텍스트 미소비) |

**핵심**: 스크립트 실행 시 코드 자체는 컨텍스트에 안 들어가고 출력만 들어감.

---

## 2. Skill 구조

```yaml
---
name: your-skill-name          # 64자 이하, 소문자+숫자+하이픈
description: 무엇을/언제 사용하는지  # 1024자 이하
---

# Skill 본문 (지시문, 예시 등)
```

```text
my-skill/
├── SKILL.md        # 필수
├── FORMS.md        # 선택 (추가 지시문)
└── scripts/
    └── process.py  # 선택 (실행 코드)
```

**name 제약**: `anthropic`, `claude` 예약어 불가 / XML 태그 불가

## 3. 플랫폼별 지원

| 플랫폼 | 사전 제작 Skills | 커스텀 Skills | 공유 범위 |
|--------|---------------|-------------|---------|
| Claude API | ✅ | ✅ | 워크스페이스 전체 |
| Claude Code | ❌ | ✅ | 개인/프로젝트 단위 |
| Agent SDK | ❌ | ✅ | 개인/프로젝트 단위 |
| Claude.ai | ✅ | ✅ | 개인만 (관리자 중앙 배포 불가) |

**크로스 서페이스 동기화 없음**: 플랫폼마다 별도 업로드 필요.

## 4. 사전 제작 Skills

| Skill ID | 기능 |
|----------|------|
| `pptx` | PowerPoint 생성/편집/분석 |
| `xlsx` | Excel 스프레드시트, 차트 |
| `docx` | Word 문서 생성/편집 |
| `pdf` | PDF 문서 생성/리포트 |

## 5. 런타임 제약 (API)

- **네트워크 접근 없음** (외부 API 호출 불가)
- **런타임 패키지 설치 불가** (사전 설치된 패키지만 사용)
- Claude Code는 네트워크 접근 가능 / Claude.ai는 설정에 따라 다름

---

## 6. 보안

신뢰할 수 없는 출처의 Skill은 **소프트웨어 설치와 동일한 위험**. 모든 번들 파일(SKILL.md, 스크립트, 이미지) 완전 감사 필요. 외부 URL 데이터 페치 Skill은 특히 위험 (의존성 변경 시 악성화 가능).

---

> [insight] Progressive disclosure의 Level 3 실행 모델은 하네스 플러그인 마켓플레이스의 핵심 패러다임이다. 스크립트 코드가 컨텍스트에 안 들어가고 출력만 들어간다는 점은 복잡한 데이터 처리 로직을 컨텍스트 비용 없이 Skills로 캡슐화할 수 있다는 의미다. 툴 정의 대신 실행 코드로 로직을 구현하는 전략이 컨텍스트 효율을 대폭 향상시킨다.

> [insight] ~100 tokens/Skill의 메타데이터 비용은 하네스의 플러그인 마켓플레이스 설계에 직접적인 영향을 준다. 100개 Skills를 설치해도 10,000 토큰밖에 안 된다. 툴 정의(tool search 없이는 수만 토큰)와 달리, Skills의 메타데이터 기반 발견은 대규모 플러그인 생태계를 컨텍스트 부담 없이 유지하는 방법을 제공한다.

> [insight] 플랫폼 간 크로스 서페이스 동기화 미지원은 하네스의 배포 전략에서 중요한 운영 부담이다. API용으로 만든 커스텀 Skills를 Claude.ai에서도 사용하려면 별도 관리가 필요하다. 하네스에서 Skills 정의를 중앙 저장소로 관리하고 각 플랫폼에 자동 배포하는 CI/CD 파이프라인이 필요하다.

> [insight] 보안 모델은 "신뢰된 출처의 Skills만 사용"을 강조하는데, 이는 하네스의 플러그인 마켓플레이스에서 서드파티 Skills 검증 체계가 필수임을 의미한다. 사용자가 외부 Sources의 Skills를 설치하는 마켓플레이스를 운영한다면, Skills 코드 감사 + 샌드박스 테스트 + 서명 검증 파이프라인이 신뢰 체계의 기반이 돼야 한다.
