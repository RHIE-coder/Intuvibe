# Multilingual Support

---

Claude는 다국어 태스크에서 영어 대비 높은 상대 성능을 유지. Zero-shot 체인-오브-쏘트 평가 기준.

---

---

## 2. 베스트 프랙티스

1. **언어 명시**: 자동 감지 가능하지만 입출력 언어를 명시하면 신뢰도 향상. "원어민처럼 자연스럽게" 프롬프트로 유창성 향상 가능
2. **네이티브 스크립트 사용**: 로마자 표기(transliteration) 대신 원어 스크립트로 입력
3. **문화적 맥락 고려**: 번역 이상의 문화·지역 맥락 인식 필요

---

## 3. 지원 범위

- 표준 Unicode 문자를 사용하는 대부분의 언어 처리 가능
- 광범위하게 사용되는 언어에서 특히 강한 성능
- 디지털 자원이 적은 언어(Swahili, Yoruba 등)도 의미 있는 수준 유지

---

> [insight] 한국어 성능이 Opus 4.1 기준 96.6%로 영어 대비 매우 높다. 하네스에서 한국어 인터페이스나 한국어 문서 처리 에이전트를 설계할 때 별도의 번역 레이어 없이 한국어 직접 처리가 충분히 실용적이다.

> [insight] 성능 편차가 언어마다 크다 (Yoruba Haiku 4.5: 52.7%). 하네스에서 다국어 지원을 제공할 경우, 저자원 언어(Swahili, Yoruba 등)는 더 큰 모델(Opus/Sonnet)로 라우팅하거나 명시적 성능 한계를 사용자에게 안내하는 로직이 필요하다.

> [insight] 모든 수치가 Extended Thinking 활성화 기준이라는 점 주목. 실제 프로덕션 하네스에서는 thinking 없는 기본 모드의 다국어 성능을 별도로 평가해야 한다. Thinking이 다국어 성능에 특히 큰 영향을 미친다면, 다국어 에이전트에서 thinking을 기본 활성화하는 것이 유리할 수 있다.
