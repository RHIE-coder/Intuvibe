# Hosting the Agent SDK

---

Agent SDK는 상태를 유지하며 명령을 실행하는 **장기 실행 프로세스** — 기존 stateless API와 다른 호스팅 고려사항 필요.

---

## 1. 시스템 요구사항

- **Python SDK**: Python 3.10+ (+ Node.js — SDK에 번들된 Claude Code CLI용, 별도 설치 불필요)
- **TypeScript SDK**: Node.js 18+
- **권장 리소스**: RAM 1GiB, 디스크 5GiB, CPU 1개
- **네트워크**: 아웃바운드 HTTPS → `api.anthropic.com` (MCP 서버 사용 시 추가)

**필수**: 샌드박스 컨테이너 환경 (프로세스 격리, 리소스 제한, 에페머럴 파일시스템)

---

## 2. 샌드박스 프로바이더

Modal Sandbox, Cloudflare Sandboxes, Daytona, E2B, Fly Machines, Vercel Sandbox  
자체 호스팅: Docker, gVisor, Firecracker (상세 격리 설정은 Secure Deployment 문서 참고)

---

## 3. 배포 패턴

| 패턴 | 설명 | 적합 케이스 |
|------|------|------------|
| **Ephemeral** | 태스크마다 새 컨테이너 생성 → 완료 후 삭제 | 버그 수정, 문서 번역, 이미지 처리 |
| **Long-Running** | 지속 실행 컨테이너, 필요 시 다중 에이전트 프로세스 | 이메일 에이전트, 사이트 빌더, 고빈도 챗봇 |
| **Hybrid** | DB 또는 세션 재개로 상태 복원 가능한 에페머럴 컨테이너 | 프로젝트 매니저, 딥 리서치, 고객 지원 |
| **Single Container** | 하나의 컨테이너에서 여러 에이전트 실행 | 시뮬레이션, 에이전트 간 협업 (에이전트 간 파일 충돌 주의) |

---

## 4. FAQ 요약

| 질문 | 답 |
|------|-----|
| 컨테이너와 통신 방법? | 컨테이너 포트 노출 → HTTP/WebSocket 엔드포인트 |
| 컨테이너 비용? | 토큰 비용이 지배적, 컨테이너 최소 ~$0.05/시간 |
| 유휴 컨테이너 셧다운 기준? | 프로바이더 의존, 예상 사용자 응답 빈도 기반으로 튜닝 |
| 세션 타임아웃? | 에이전트 세션 자체는 타임아웃 없음 → `maxTurns` 설정으로 루프 방지 |
| CLI 업데이트 빈도? | semver 버저닝 — 브레이킹 체인지는 메이저 버전 |

---

> [insight] 4가지 배포 패턴 중 하네스의 플러그인 마켓플레이스에는 **Hybrid 패턴**이 가장 적합하다. 사용자가 플러그인을 실행할 때만 컨테이너를 스핀업하고, SDK 세션 재개(`resume=session_id`)로 이전 상태를 복원하면 비용 효율과 연속성을 동시에 달성한다. 장기 실행 플러그인(리서치, 모니터링)은 Long-Running 패턴으로 별도 분류하는 것이 타당하다.

> [insight] 컨테이너당 권장 리소스(1GiB RAM, 5GiB 디스크, 1 CPU)는 하네스의 플러그인 실행 환경 스펙 정의의 기준값이다. 플러그인 유형별(파일 처리 집중형, 네트워크 집중형, 계산 집중형)로 다른 리소스 프로파일을 마켓플레이스 스펙에 포함하면 운영 비용 예측이 가능해진다.
