# Extend Claude Code — 확장 레이어 구조

---

Claude Code의 빌트인 도구(파일, 검색, 실행, 웹)는 에이전틱 루프의 기본 능력이다.
이 문서는 그 위에 얹는 **확장 레이어** — 6가지 확장 기능과 패키징 단위인 Plugin을 다룬다.

---

## 1. 확장 기능 6종 — 에이전틱 루프에 꽂히는 위치

각 기능은 에이전틱 루프의 서로 다른 지점에 개입한다.

| 기능 | 루프 내 위치 | 한줄 정의 |
|------|------------|----------|
| **CLAUDE.md** | 루프 진입 전 (항상 주입) | 매 세션 자동 로드되는 영구 지침서 |
| **Skill** | 루프 내부 (온디맨드 주입) | 재사용 가능한 지식/워크플로우 마크다운 파일 |
| **Subagent** | 루프 바깥 (독립 루프 생성) | 자체 컨텍스트에서 돌고 요약만 반환하는 격리 워커 |
| **Agent Teams** | 루프 바깥 (다중 독립 루프) | 여러 세션이 공유 작업 목록 + 상호 메시징으로 협업 |
| **MCP** | 루프 내부 (도구 확장) | 외부 서비스를 도구로 연결하는 프로토콜 |
| **Hook** | 루프 바깥 (이벤트 콜백) | 특정 이벤트에 반응하는 결정론적 스크립트. LLM 불관여 |

**Plugin** = 위 기능들을 묶어 하나의 설치 단위로 배포하는 패키징 레이어. 네임스페이스(`/plugin:skill`)로 충돌 방지.

---

## 2. 기능 간 경계 — 뭘 언제 쓰는가

### 지식/지침 전달 계층: CLAUDE.md → Rules → Skill

```
항상 적용         조건부 적용         필요 시 적용
CLAUDE.md    →   .claude/rules/   →   Skill
(매 세션)        (매칭 파일 열릴 때)   (호출 시)
전체 스코프       경로별 스코프         작업별 스코프
```

- **CLAUDE.md** — "항상 X 해라" 규칙. **200줄 이하** 유지가 원칙
- **Rules** (`.claude/rules/`) — `paths` frontmatter로 파일 경로 스코핑. 언어별/디렉토리별 가이드라인
- **Skill** — 참조형(API 가이드) 또는 액션형(`/deploy`). `@path` 임포트로 파일 포함 가능

> [insight] Skill은 세션 시작 시 **설명(description)이 매 요청에 포함**된다. Skill 수가 늘면 설명만으로도 시작 비용이 누적된다. 호출 시에는 전체 내용이 메인 컨텍스트에 합류하며, 마크다운 참조 링크로 걸린 파일은 Claude가 필요 시 온디맨드로 읽는다. 대응: 수동 전용 Skill은 `disable-model-invocation: true`로 설명조차 올리지 않기, 무거운 참조 Skill은 Subagent에서 `context: fork`로 격리 실행, Skill 설명은 짧고 명확하게 유지.
>
> [insight] `@path` 임포트는 CLAUDE.md와 Rules에서만 동작하는 **자동 확장** 문법이다. 세션 시작 시 참조된 파일 내용이 컨텍스트에 무조건 삽입되며, 최대 5단계 재귀까지 지원한다. 경로는 작업 디렉토리가 아닌 **해당 파일 기준 상대경로**로 해석된다. Skill에서는 `@path`가 아닌 마크다운 링크로 참조하며, Claude가 판단해서 읽는 온디맨드 방식이다. — CLAUDE.md/Rules에서 `@path`를 남발하면 세션 시작부터 컨텍스트가 폭발할 수 있다.

### 실행 위임 계층: Skill → Subagent → Agent Team

```
같은 컨텍스트      격리 컨텍스트       완전 독립 + 상호 통신
Skill          →  Subagent        →  Agent Team
(메인 윈도우)     (요약만 반환)       (팀원끼리 직접 메시지)
```

| 판단 기준 | Skill | Subagent | Agent Team |
|----------|-------|----------|------------|
| 컨텍스트 | 메인에 합류 | 격리, 요약만 반환 | 완전 독립 |
| 통신 | - | 메인에만 보고 | 팀원 간 직접 메시지 |
| 토큰 비용 | 메인에 누적 | 낮음 (요약만) | 높음 (각각 별도 인스턴스) |
| 전환 시점 | - | 컨텍스트 격리 필요 시 | subagent 간 통신이 필요하거나 컨텍스트 한계 도달 시 |

조합: Subagent에 Skill 프리로드(`skills:` 필드), Skill에서 `context: fork`로 격리 실행

### 외부 연결: MCP vs Skill

- **MCP** = 외부 서비스에 대한 **연결**(도구와 데이터 접근)
- **Skill** = 그 연결을 **잘 쓰는 법**(스키마, 쿼리 패턴, 포맷 규칙)
- 조합 예: MCP로 DB 연결 + Skill로 데이터 모델/쿼리 패턴 교육

## 3. 레이어링 — 같은 이름이 여러 레벨에 있을 때

| 기능 | 충돌 해소 방식 | 우선순위 |
|------|-------------|---------|
| **CLAUDE.md** | additive (모두 합산) | 충돌 시 더 구체적인 것 우선 (Claude 판단) |
| **Skills** | 이름 기준 override | managed > user > project |
| **Subagents** | 이름 기준 override | managed > CLI flag > project > user > plugin |
| **MCP 서버** | 이름 기준 override | local > project > user |
| **Hooks** | merge (전부 실행) | 매칭 이벤트에 등록된 모든 훅이 발화 |

## 4. 컨텍스트 비용 — 하네스 설계 시 핵심 제약

```
세션 시작 시 로드                        세션 중 로드              루프 밖
─────────────────────────────────      ──────────────────────    ──────────
CLAUDE.md    → 전체 (매 요청 포함)      Skill 호출 → 전체 내용    Hook → 0
MCP          → 도구 이름만 (deferred)   Subagent   → 격리 컨텍스트
Skills       → 이름+설명만
```

| 기능 | 로딩 시점 | 평소 비용 | 사용 시 비용 |
|------|----------|----------|------------|
| **CLAUDE.md** | 세션 시작 | **매 요청** (전체 내용) | - |
| **Skill** | 세션 시작(설명만) | 낮음 | 호출 시 전체 로드 |
| **MCP** | 세션 시작(이름만) | 낮음 | 도구 사용 시 스키마 로드 |
| **Subagent** | 스폰 시 | 0 (격리) | 자체 컨텍스트 소비, 메인에 영향 없음 |
| **Hook** | 트리거 시 | 0 | 0 (출력 반환 시만 추가) |

**비용 절감 수단:**
- Skill에 `disable-model-invocation: true` → 수동 호출 전까지 비용 제로 (설명도 안 올라감)
- MCP tool search가 기본 ON → 유휴 MCP 도구는 최소 비용
- Subagent로 위임 → 메인 컨텍스트 부풀림 방지
- `/mcp`로 서버별 비용 확인, `/context`로 전체 사용량 확인

> [insight] MCP 도구와 Skill은 **이름이 곧 검색 키**다. tool search가 이름 기반으로 후보를 매칭하고, 선택된 후보만 전체 스키마를 로드하는 2단계 구조(이름으로 후보 선정 → 스키마로 상세 파악). 이름이 모호하면 Claude가 도구를 후보로 올리지 못하거나 엉뚱한 것을 선택할 수 있다. 원문: *"If descriptions are vague or overlap, Claude may load the wrong skill or miss one that would help."* — 도구/스킬 네이밍 컨벤션을 초기에 확립해야 한다.

## 5. 조합 패턴 요약

| 패턴 | 구조 | 예시 |
|------|------|------|
| **Skill + MCP** | MCP = 연결, Skill = 사용법 | DB 연결 + 스키마/쿼리 패턴 문서 |
| **Skill + Subagent** | Skill이 병렬 Subagent 스폰 | `/audit` → 보안/성능/스타일 워커 동시 실행 |
| **CLAUDE.md + Skill** | 상시 규칙 + 온디맨드 참조 | "API 규칙 따라라" + API 스타일 가이드 전문 |
| **Hook + MCP** | Hook → MCP로 외부 액션 | 파일 편집 → Slack 알림 |

> **MCP 주의사항:** 연결이 세션 중 끊어져도 경고 없이 도구가 사라진다. `/mcp`로 상태 확인 필요.
