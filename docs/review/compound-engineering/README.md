## 이 묶음의 의미

Compound Engineering은 현재 review 대상들 중에서 가장 **지식 축적 지향적**이다.

특히 핵심 축은 다음과 같다.

- brainstorm → plan → work → review → compound
- right-size ceremony
- 28 reviewer persona 기반 review system
- `docs/solutions/`를 통한 해결책 축적
- 플랫폼 변환 CLI까지 포함한 배포 구조

즉 이 묶음은 "강제 워크플로우"보다, **작업이 다음 작업을 더 쉽게 만들도록 지식을 누적하는 구조**를 보는 비교군이다.

---

## 하네스 관점의 한 줄 정리

> Compound Engineering은 workflow 자체보다 "지식이 누적되는 개발 체계"를 강하게 밀어붙이는 플러그인이다.
> 현재 하네스에서 `bench`, `book`, `design`, `solutions` 류 자산을 어떻게 연결할지 생각할 때 중요한 대조군이다.
