# Performance Review Reference

## 검사 항목

### 알고리즘 복잡도

- O(n^2) 이상 루프 (중첩 반복, 불필요한 전체 탐색)
- 정렬 후 검색 대신 해시맵 미사용
- 대량 데이터에 선형 탐색

### N+1 쿼리

- 루프 안에서 DB 쿼리 (ORM eager loading 누락)
- 연관 데이터 개별 fetch (batch/join 미사용)

### 메모리 누수

- 이벤트 리스너 미해제
- 순환 참조 (WeakRef 미사용)
- 대용량 객체 캐시 무한 성장 (LRU/TTL 미적용)
- 클로저에 의한 의도치 않은 참조 유지

### 불필요 할당

- 루프 내 반복 객체 생성
- 문자열 연결 대신 배열 join 미사용
- 대량 데이터 복사 (spread/slice 남용)

### I/O 패턴

- 동기 파일 I/O (대용량 파일)
- 직렬 API 호출 (병렬화 가능한 경우)
- 커넥션 풀 미사용

## 판단 기준

- **BLOCK**: 프로덕션 장애 수준 (무한 루프, 메모리 폭발)
- **NEEDS_CHANGE**: N+1, O(n^2) 루프, 메모리 누수 패턴
- **PASS**: 성능 안티패턴 미발견
